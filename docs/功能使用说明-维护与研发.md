# 5-15新增功能说明（零星工程 · 维护管理 · 人员在岗 · 研发管理）

> 适用版本：近期迭代（维护排单、零星工程、人员在岗状态、研发待办/研发文档）。  
> 入口均在管理后台左侧菜单；具体菜单是否可见取决于账号权限。

---

## 一、维护排单

**菜单路径**：维护管理 → **维护排单**

### 1.1 列表新能力

| 功能 | 怎么用 |
|------|--------|
| **列设置** | 点击工具栏 **列设置**，勾选要显示的列（排单号、标题、类型、截止时间、**工程金额**、进度、状态、执行人、施工人员、审批等）。**操作**列始终显示。可点 **恢复默认（全部显示）**。 |
| **表头排序** | 点击可排序列的表头，在 **升序 / 降序** 间切换。若隐藏了当前排序列，排序会自动取消。 |
| **施工人员列** | 展示该任务各施工日人员的 **并集**（仅姓名）；未指定时显示 **—**。 |
| **状态筛选** | 工具栏 **全部 / 未完成 / 已完成 / 过期完成** 切换列表范围。 |
| **导出 Excel** | 点击 **导出 Excel**，按当前关键词与状态筛选导出（单次最多 2000 条）。 |

**搜索**：支持排单号、标题、内容、执行人、截止时间、**工程金额**。

### 1.2 推荐办理流程

1. **新建任务**（可选填 **工程金额**，留空按 0 处理）  
2. 点 **办理** → 在 **分配与施工** 区 **分配任务**（指定 **执行人**；之后可 **转派**）  
3. 可选：在 **施工人员（按日）** 按自然日补充人员与工时  
4. **提交钉钉审批**（若已启用门禁）  
5. 审批通过后 **追加操作记录**（进度每次至少 +10%）  
6. 抽屉右上角点 **完结** 办结  

### 1.3 办理抽屉（分区说明）

打开 **办理** 后，内容分为三块，自上而下：

#### ① 分配与施工

- **分配任务 / 转派任务**：选择 **执行人**（在职系统用户）并确认。  
- **施工人员（按日）**（须已分配执行人）：
  - 表格列：**施工日期**、**工时**（半天 / 整天 / 加班）、**施工人员**、**操作**。
  - 下方表单选日期、工时类型、施工人员后点 **保存该日施工人员**。
  - 同一日可有多条记录；要改已有行请先在表格点 **填入表单**，再保存。
  - 可 **删除** 单条施工记录。

**任务已完成 / 已取消 / 审批中** 时，施工人员区为 **只读**；若无按日记录，会按 **施工人员（汇总）** 合成一行展示。

#### ② 任务概况

只读查看：状态、类型、截止时间、**工程金额**、进度、执行人、**施工人员（汇总）**、任务说明、钉钉审批状态等。若任务 **超时后完结**，状态显示 **过期完成**，并展示 **过期完成时间**。

#### ③ 操作记录

- 时间线展示分配、转派、按日施工人员、操作说明、逾期等记录。  
- 未完结时可 **追加操作记录**：填写说明、可选附图、拖动 **更新进度（%）**，点 **保存记录**。保存后「已排单」可变为「执行中」。

### 1.4 完结

- 入口：办理抽屉右上角 **完结**（非列表）。  
- 须已分配执行人；若启用钉钉审批，须 **审批通过**。  
- 弹窗 **确认完结** 中会预览施工人员表格（优先按日；无按日数据时显示汇总一行）。  
- 可填 **办结说明（可选）**；确认后状态为 **已完成**，进度 100%。  
- 若完结时刻 **晚于截止时间**，状态标记为 **过期完成**（仍计入「已完成」筛选），并记录 **过期完成时间**。

## 二、零星工程

**菜单路径**：维护管理 → **零星工程**

### 2.1 业务流程（五步）

```
① 分配到部门 → ② 按日分派施工人员 → ③ 确认派单 → ④ 跟踪与说明 → ⑤ 闭环
```

抽屉顶部 **步骤条** 与 **工程概况 / 办理** 两个大分区，便于按阶段操作。

| 阶段 | 主要操作 |
|------|----------|
| 待派单 | **① 分配到部门** 指定部门负责人 → **② 按日分派施工人员** → **③ 确认派单**（计划完成时间、派单说明、图片） |
| 已派单 / 执行中 | **转派 · 部门负责人**、**转派 · 按日施工人员**、**④ 部门负责人跟踪与说明** |
| 已闭环 | 概况与按日表 **只读**；时间线含 **闭环** 记录 |

### 2.2 按日施工人员

与维护排单类似：

- **工时类型**：半天、整天、加班。  
- 同一自然日可多条；**填入表单** 后修改已有条目；不带条目保存可 **新增** 同日另一条。  
- 人员须为 **已绑定钉钉 userId** 的在职用户。  
- **闭环前** 须在至少一个施工日指定人员（列表 **施工人员** 列为各日并集）。

### 2.3 跟踪与闭环（含新增：闭环自动写跟踪）

**④ 部门负责人跟踪与说明**：

- 查看时间线（分配、转派、按日人员调整、手动跟踪等）。  
- 填写 **本次跟踪说明**、**更新后进度（%）**，点 **保存跟踪记录**。首次保存后「已派单」→「执行中」。

**确认闭环**（抽屉右上角）：

- 须至少一日有施工人员；若启用钉钉审批须已通过。  
- 弹窗可填 **闭环说明（可选）**。  
- **闭环成功后**，系统会 **自动在时间线追加一条「闭环」类型记录**（含完成日期与说明），无需再手动补一条跟踪。  
- 进度记为 100%，**实际完成** 日期写入概况。  
- 若闭环时刻 **晚于截止时间**，状态标记为 **过期完成**，并展示 **过期完成时间**。

### 2.4 列表与其它

- **列设置**、表头排序（与维护排单同类）。  
- **状态筛选**：**全部 / 未闭环 / 已闭环 / 过期完成**。  
- **导出 Excel**：按当前筛选导出（单次最多 2000 条）。  
- **办理** 打开分区抽屉；**编辑** 改基本信息；**提交钉钉审批** / **删除** 受审批状态限制。

## 三、人员在岗状态

**菜单路径**：**人员在岗状态**（独立一级菜单，路径 `/personnel-presence`）  
**权限**：需具备维护相关权限（`maintenance`）。

### 3.1 功能说明

以 **团队周历** 形式展示成员每日在岗情况。成员名单与 **用户管理** 同步（在职且已绑定钉钉；排除机器人等功能账号）。

### 3.2 如何使用

| 操作 | 说明 |
|------|------|
| **上一周 / 下一周 / 本周** | 切换查看周次 |
| **筛选成员** | 按姓名缩小列表 |
| **刷新** | 重新加载数据 |
| 点击本人行的日期格 | 弹出 **选择当日状态**：**在岗**、**休息**、**出差**、**请假** |
| 编辑说明 | 可为当日填写说明（最多 500 字，可空） |

### 3.3 规则与权限

- **每人只能编辑自己那一行**；他人行为 **只读**。本人行带 **我** 标识。  
- **工作日** 未设置时，默认 **在岗**。  
- **周日** 未设置时，默认 **休息**（表头带 **默认休** 提示）。  
- 当天列会高亮显示。

## 四、研发管理

**菜单路径**：研发管理 → **研发待办** / **研发文档**  
**权限**：`rd-mgmt`

### 4.1 研发待办

**路径**：`/rd/todos`

#### 列表

- 分段：**进行中** / **已完成** / **全部**。  
- 搜索：主题、内容、备注、负责人。  
- 列：主题、具体内容、状态、负责人、截止、更新、主图、备注、操作。  
- 点击行或 **查看详情** 打开右侧抽屉。

#### 新建 / 编辑

| 字段 | 说明 |
|------|------|
| **主题名称** | 必填 |
| **具体内容** | 必填；支持在框内粘贴截图 |
| **负责人员** | 多选 |
| **截止日期时间** | 可选；不选表示无截止 |
| **备注** | 纯文本 |
| **图片附件** | 上传前浏览器压缩 |

#### 详情与跟踪

- **完成** / **重新打开** 会 **自动写入系统跟踪**（无需手填）。  
- **新增跟踪**：填写 **跟踪说明**，可附图（最多 3 张），点 **保存跟踪记录**。  
- 用户跟踪可 **删除**；系统跟踪为只读展示。

### 4.2 研发文档

**路径**：`/rd/docs`  
**阅读页**：`/rd/docs/preview/:docId`（编辑区 **阅读预览** 或文档 **预览**）

#### 文件夹与列表

- 左侧 **文件夹树**：**新建**、**重命名**、**移动**、**删除**（须为空文件夹）。  
- **全部文档**：不按文件夹过滤查看全库。  
- 右侧 **文件与文档** 表：**预览**、**编辑**（富文本文档）、**下载**、**移动**、**删除**。  
- **本页不提供新文件上传入口**；已有附件类文件可管理，新建内容以 **富文本文档** 为主。

#### 新建 / 编辑文档

1. 在左侧选中文件夹，点 **新建文档**。  
2. 填写 **标题**，在 **正文编辑** 区编写（工具栏：标题、字号、加粗、列表、链接、**插入图片** 等）。  
3. 点 **保存** 才真正创建记录（保存前列表无条目）。  
4. 保存时可填 **修改说明**（可选）；留空记为常规保存。  
5. **阅读预览** 在新窗口打开带目录的阅读页。

#### 存储提示

页顶展示 **全系统附件剩余** 配额；接近或超过上限时会有提示（仍可按策略继续保存，以系统配置为准）。

## 五、钉钉审批（维护排单 / 零星工程共用说明）

若单位启用了 **钉钉审批门禁**：

- 须先 **分配** 执行人（排单）或部门负责人（零星工程）。  
- 在列表或办理抽屉 **提交钉钉审批**，在钉钉中处理流程。  
- **审批通过** 后，方可：零星工程 **确认派单**、上传派单图、**保存跟踪**、**确认闭环**；维护排单 **追加操作记录**、附图、**完结**。  
- **审批中** 不可编辑基本信息或删除（以页面按钮灰显为准）。

## 六、常见问题

| 问题 | 处理建议 |
|------|----------|
| 维护排单完结弹窗没有施工人员表 | 确认是否已分配执行人并有按日或汇总施工人员；完结弹窗会单独拉取最新数据。 |
| 已完成任务看不到按日施工表 | 在办理抽屉 **分配与施工** 区查看 **只读** 按日表（概况里仅有汇总一行文字）。 |
| 零星工程无法闭环 | 检查是否至少一个施工日有人员；钉钉是否已通过；状态是否为已派单/执行中。 |
| 闭环后时间线没有说明 | 近期版本会在闭环时 **自动追加「闭环」跟踪**；重新打开办理抽屉即可看到。 |
| 人员在岗改不了别人 | 设计如此，仅可编辑 **本人** 行。 |
| 研发文档找不到上传文件 | 本模块以在线文档为主；请用 **新建文档** + 富文本 **插入图片**，而非上传独立文件。 |
